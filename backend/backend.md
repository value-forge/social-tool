# Backend 项目说明

## 项目简介
社交媒体工具后端服务，用于抓取 Twitter 推文并通过 AI 分析给出评论建议，推送到钉钉。

## 钉钉推送格式

当 AI 分析完成推文后，会发送如下格式的消息到钉钉（紧凑无换行版）：

```
📱 推特：@{作者用户名} | 🕐 {发布时间}

📝 {推文原文}

💬 {AI总结}

⭐ 评分：{评分}/10 | 💡 {评论建议}

👍 {点赞数} 💬 {回复数} 🔄 {转发数}

🔗 {推文链接}
```

### 字段说明

| 字段 | 说明 |
|------|------|
| 作者用户名 | Twitter 用户的 @handle |
| 发布时间 | 推文发布时间（格式：MM-DD HH:mm）|
| 推文原文 | 完整的推文内容（单行显示）|
| AI总结 | Kimi AI 生成的内容摘要 |
| 评分 | 1-10 分，评估话题热度、评论空间、涨粉潜力 |
| 评论建议 | AI 给出的具体评论建议 |
| 点赞数/回复数/转发数 | 推文的互动数据 |
| 推文链接 | 点击跳转到原推文 |

### 示例

```
📱 推特：@XXY177 | 🕐 03-26 09:29

📝 以为是德高望重的长者，结果是位绅士君子 你们没发现吗，一般长寿的，都是老人

💬 这是一条幽默类推文，用反转手法制造笑点，适合轻松互动

⭐ 评分：7.0/10 | 💡 可以从幽默角度回应，比如"长寿的秘诀是保持绅士风度吗😄"

👍 42 💬 8 🔄 5

🔗 https://x.com/XXY177/status/2037341207186243976
```

## 技术栈

- Go 1.24
- MongoDB
- OpenClaw ACP (Kimi AI)
- DingTalk Webhook

## 运行方式

```bash
# 启动 MongoDB
docker-compose up -d

# 编译并运行 Worker
./worker
```

## 配置项

通过环境变量配置：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| MONGODB_URI | mongodb://admin:password@localhost:27017/social_tool | MongoDB 连接字符串 |
| MONGODB_DB | social_tool | 数据库名 |
| DINGTALK_WEBHOOK | - | 钉钉机器人 Webhook 地址 |
| FETCH_INTERVAL | 5m | 抓取间隔 |
| LOG_LEVEL | info | 日志级别 |
