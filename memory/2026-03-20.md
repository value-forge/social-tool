### Tech Stack
- **Backend**: Go + Kratos microservices framework

- **Database**: MongoDB (Docker)

### Features Implemented Today

1. **Twitter OAuth 2.0 Login**
   - PKCE flow implementation
   - JWT token management
   - Automatic platform connection creation

2. **Mock Mode** (Free, for development)
   - Environment: `TWITTER_MOCK_MODE=true`
   - Simulated following list with 5 accounts (Elon, Naval, etc.)
   - Simulated tweets data
   - No Twitter API costs

3. **bird-cli Integration** (Free alternative to Twitter API)
   - bird-cli is a command-line tool (NOT a browser plugin)
   - Uses browser cookies (Chrome/Safari/Firefox) for auth
   - No API key needed, reads public data
   - Environment: `TWITTER_USE_BIRD=true`
   - Endpoints:
     - `GET /api/bird/following?username={user}`
     - `GET /api/bird/posts?username={user}`
   - Safety: Read-only operations, relatively safe for personal use

4. **Following List UI**
   - Dashboard page with sidebar menu
   - Following list panel with table view
   - Displays avatar, name, bio, follower count
   - Links to X profiles

### API Endpoints

| Endpoint | Auth | Description |
|----------|------|-------------|
| `GET /api/twitter/following` | JWT | Get following (Mock/API mode) |
| `GET /api/twitter/posts` | JWT | Get tweets (Mock/API mode) |
| `GET /api/bird/following` | None | Get following via bird-cli |
| `GET /api/bird/posts` | None | Get tweets via bird-cli |
| `GET /api/auth/twitter/url` | None | OAuth URL |
| `POST /api/auth/twitter/callback` | None | OAuth callback |

### Configuration

```bash
# Mock mode (recommended for development)
TWITTER_MOCK_MODE=true ./social-tool-server

# bird-cli mode (free real data)
TWITTER_USE_BIRD=true ./social-tool-server

# With default username for bird-cli
TWITTER_USERNAME=0xpeterlee TWITTER_USE_BIRD=true ./social-tool-server
```

### Twitter API Pricing (Reference)
- **Free**: 1,500 tweets/month, NO following endpoint
- **Basic**: $100/month, 10,000 tweets/month, includes following
- **Pro**: $5,000/month

### bird-cli Commands (for reference)
```bash
# Check login
bird whoami

# Get user tweets
bird user-tweets 0xpeterlee -n 50 --json

# Get following
bird following --json

# Get followers
bird followers --json
```

- bird-cli is NOT a plugin - it's a CLI tool that can be called from code via exec.Command
- For production use, need either Twitter API Basic ($100/mo) or accept bird-cli limitations
- Mock mode is sufficient for UI development and demos
