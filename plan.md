## 可补充方向（涨粉 / 曝光）

6. **发帖节奏与「最佳时段」**：按粉丝活跃时区推荐发布时间；支持草稿队列与定时发，避免扎堆或断更。
7. **内容形态**：Thread 长文拆解、投票、引用推文串、图文/外链卡片优化预览，提高停留与转发率。
8. **话题与趋势雷达**：订阅关键词/榜单变化，在「刚起势未刷屏」时介入，比纯蹭旧帖更有效。
9. **互动质量而非数量**：高互动粉丝/潜在 KOL 列表、互动冷却提醒，减少无效机械回复被判定为骚扰。
10. **列表与社群**：维护公开 List、参与垂直 Space/直播预告，把「路人」导到固定触达渠道。
11. **数据复盘**：单帖曝光-互动漏斗、粉丝净增与来源（哪类内容带来关注）、掉粉预警。
12. **人设与栏目化**：固定栏目（周报、资源合集）、置顶与简介联动，让新访客一眼知道关注理由。
13. **合规与人审开关**：批量/自动操作加频率上限、随机延迟、敏感词与 API 限额提示，降低封号风险。

> 说明：X/Twitter 对自动化、垃圾互动有规则限制，工具侧宜强调「辅助决策 + 人工确认」，避免纯机器刷屏式运营。

## 冷启动阶段（大致 0～几百粉）

**目标**：先让「点进主页的人」愿意关注，再争取少量转发带来的外溢曝光；不追求频率轰炸。

### 账号门面（先做足再拉流）
- **头像 + 昵称 + 简介**：一句话说清领域与价值（关注你能持续得到什么）。
- **置顶**：1 条代表作或「新人从这读起」的短 thread / 链接树（Linktree 等）。
- **背景图**：可选，强化人设或代表作视觉。

### 内容（少而稳）
- **固定频率**：例如每周 3～5 条有效内容，比每天灌水更重要。
- **优先 thread / 清单体**：「10 个要点」「3 步上手」类比短碎碎念更易被收藏转发。
- **一个主垂类**：冷启动期忌话题太散，算法与路人都难形成记忆。

### 曝光从哪里来（没有粉丝时）
- **去「有流量的帖」下做高质量回复**：补充数据、反例、步骤，争取被顶到前几条（比发很多原创更有效）。
- **带 1～2 个精准话题/关键词**（忌堆砌标签）。
- **其他平台同名导流**：GitHub、博客、即刻、Telegram 频道简介里写死 X 主页。

### 互动（克制）
- **每天主动互动有上限**：关注/点赞/回复都别太像机器；回复要有信息增量。
- **找「体量略大于你」的账号**互动，偶尔能带来回流；忌只蹭顶流评论区淹没。

### 冷启动阶段少做或慎做
- **大规模互关、批量取关、自动化刷屏**：容易伤号或被判垃圾行为。
- **纯 AI 无编辑堆帖**：易被识别为低质，损害信任。

### 最小复盘（哪怕手写）
- 每周看：**哪一条主页点击率 / 互动相对高**，下周多做同类选题。
